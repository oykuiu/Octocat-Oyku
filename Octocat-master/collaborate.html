Collaborate

In this exercise you're going to collaborate with a partner using GitHub. You'll both work on the same project and see what happens if your changes conflict

Add a collaborator to your repository

If you want to work together with someone on a project you will need to add them as a collaborator. By default, only you can make changes to repositories you own. By adding a collaborator you're allowing somebody else to push to the repository. You can only add collaborators to your repositories in the online web-based version of GitHub.

  1. Find somebody to pair up with. Each of you will add the other to your GitHub repositories
  2. Go to your online GitHub account and select the repository that you want to add a collaborator to.
  3. Now click on the settings tab for the repository and then click on the collaborators tab.
  4. Search for the username of the person you want to add to your repository, once you have found the person click on add as collaborator
  5. This will send an invitation email to your collaborator's address. Ask them to check their email and accept.

Cloning a workspace

We've already looked at how to create a new repository from scratch. But what about when someone adds us to a project. The repository is already there; we just want to create a new workspace. In GitHub this is known as cloning a repository.

  1. Once you've accepted the invitation, open up GitHub Desktop app.
  2. Go to File -> CLone Repository...
  3. If the new repository doesn't show up in the list you may need to fully close and re-open the GitHub desktop app.
  4. Choose the repository you want to clone and take note of the Local Path. This is where you'll find the new workspace on your computer.
  5. Make a change to this repository, commit and push. Ask the repository owner to sync their workspace. Make sure they can see your changes.

Set up a conflict scenario

Conflicts arise when you have multiple parties adding code to a shared repository, where one party is working on an old version of the code that has been updated.

  1. Ensure you and your collaborator are both looking at the latest version of your workspace.
  2. Change the following piece of text to your name ***JACK***
  3. Commit and sync your changes. Your collaborator should not yet sync
  4. Ask your collaborator to change the text to her name. Then try to commit and sync

Fix a conflict

Whoever is last to sync for a conflict has to fix it, unfortunately!

  1. The conflicts will have lines around them like this <<<<<<<<<< And the two versions will be separated with marks like this ========.
  2. Choose which version you want to keep and delete the other lines
  3. Edit the lines if necessary.
  4. Remove the lines with arrows <<<<<<<<<< and ===========.
  5. Add and then commit and push the changes.
