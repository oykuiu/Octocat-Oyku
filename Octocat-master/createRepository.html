Create a Local Repository

In this exercise you're going to create your own repository on GitHub. We'll be using these files to create your first project.

Create an empty repository

First thing's first; we need to create an empty repository so we can start work on our project. In this step we're going to use GitHub Desktop to create an empty repository and a workspace, and make sure we can find the workspace on our computer.

  1. In your GitHub Desktop app click on File -> New Repository...
  2. On the =create tab type a name for your repository (note you name has to have a hyphen instead of spaces (e.g. cfg-project-1). Before you press create repository make a note of the local path as this shows where to find the code folder on your computer.
  3. When you're happy, click on Create Repository. Then find and open the code folder on your computer. You're new repository is ready to go!

Add files to the workspace

Great! You've created a new workspace on your local machine. But we haven't done anything with it yet. In this step we're going to add some files into our workspace and commit them.

  1. Open up this website's root folder in Finder / File Explorer and copy all of the files into the new folder you just created (it should be at local path from the last step
  2. Open up your GitHub Desktop application. You should see a list of all of the new files you have copied in. We're going to commit (finalize) these changes, now. Make sure the checkbox to the left of each file is ticked.
  3. Fill out a very short description (max 72 characters) of what you've just done in the textbox called Summary. You need to do this every time you commit files.
  4. It's good practice to fill out the Description textbox, too. The description can be as long as you like so you can use it when the summary doesn't give you enough space.
  5. When you're happy with the description and summary click the button labelled commit to master. You've now saved your commit **locally** (only on your computer)

Push your Changes

Now you've committed your files, but you haven't sent them up to the cloud yet. The final step is to publish (or push your changes so they're there for all the world to see!

  1. In order to make sure everyone can see your changes we need to *publish* (or *push*) our commit. Once you've committed your changes, you should see a button at the top of the GitHub client Window saying push origin.]
  2. To make sure your changes have been published, check that the button you have just clicked has changed to say Fetch origin. This means there's nothing else to push.
  3. Congratulations! You've just published your first commit! Log in to GitHub using your web browser and make sure you can see your repository online.
