Undo and Revert Commits

In this exercise you're going to see how to undo and revert commits in GitHub

Undo a commit

If you commit your changes locally and realise that you want to undo them before you push it to the remote server, you can undo your commit. This gets rid of the commit as if it never happened.

  1. Look for the <title> element on this page (use ctrl-F or cmd-F to help you find it).
  2. Change the title to your name
  3. Open up GitHub Desktop and make sure you're looking at the changes tab.
  4. Add a commit message and click commit (Do not push to the server yet)
  5. Click on the Undo button at the bottom of the screen.
  6. The commit is removed and the changes show up in the file list again.
  7. If you want to undo the changes entirely, right-click on the file and select Discard changes...

Revert a commit

If you have already pushed your changes to the server you can't undo your commit any more. You need to create a revert. In this exercise we'll look at how to do both of these.

  1. Add an <li> tag to the nav at the top of this page. Make it a link which shows your name as the text and links back to "index.html"
  2. Remove the <li> tag containing the text example 1 from the createRepository.html page
  3. Create a new file called extra.html and save it in the root folder of the website
  4. Delete the index.html file
  5. Commit the changes and publish them to the server
  6. Open up the history tab and find your commit in the list. Right click and select Revert this Commit
  7. A new commit should appear in the history list, cancelling out the commit you just made. Check that all the changes mirror each other.
